# Handoff — 2026-06-03

## Session Summary

Diagnosed and fixed the IJS pulse generation pipeline, which had been silently broken since
May 29. The launchd job (`com.jpurnell.ijs-pulse`) was unloaded and not firing. The
`generate-pulse.sh` script had no corpus sync step (telemetry files written to disk were
never committed/pushed), no API key in the launchd environment, and no health check.

## What Was Broken

1. **Launchd job unloaded**: `com.jpurnell.ijs-pulse.plist` existed on disk but was not
   registered with launchd. No pulse had been generated since May 29 (5 missed days).

2. **No corpus sync in the pipeline**: `quality-gate --check` writes telemetry JSON to the
   corpus directory via `TelemetryWriter`, but never calls `CorpusManager.sync()` to git
   commit/push. That sync only lived in `org-judgement-system`'s `ijs-telemetry push`
   command (Push.swift:95-109), which is a separate binary. 87 files from today's work
   were sitting uncommitted.

3. **Missing ANTHROPIC_API_KEY**: Launchd jobs don't source `.zshrc`, so the Keychain
   lookup (`security find-generic-password`) was never called. LLM narrative generation
   failed silently, falling back to template — but narrative durability (Refine.swift:70-119)
   correctly preserved the existing W22 LLM narrative.

4. **Health check compared wrong week**: The script checked for the *current* ISO week
   (W23), but `ijs-telemetry refine` generates the *previous complete* week (W22).

## What Was Fixed

### generate-pulse.sh (org-judgement-system/Scripts/)

Rewritten from a 20-line "run refine" script to a 5-step resilient pipeline:

- **Step 1** (every run): Pull remote, abort any stuck rebase, commit pending telemetry,
  push to remote.
- **Step 2** (first run of the day): Generate pulse via `ijs-telemetry refine --narrate`.
  Uses a marker file (`.pulse-generated-YYYY-MM-DD`) to skip on subsequent runs.
- **Step 3**: Commit and push pulse output.
- **Step 4**: Health check — finds latest pulse on disk, reports size/age, confirms
  narrative type (LLM vs template).
- **API key**: Loaded from macOS Keychain via `security find-generic-password` when
  `ANTHROPIC_API_KEY` is not already set in environment.

### com.jpurnell.ijs-pulse.plist (~/Library/LaunchAgents/)

- Changed from once daily (8am) to **every 2 hours** (00, 02, 04, ..., 22).
- Added `RunAtLoad: true` so the job fires immediately after login/reboot.
- Added `EnvironmentVariables` with `PATH` including `/usr/local/custom/bin`.
- Job is now loaded and verified running.

## Narrative Durability Recap

The priority chain (Refine.swift:70-119):

```
new LLM > preserved LLM > preserved template > fresh template
```

A successful LLM call always replaces what's on disk. A *failed* LLM call never overwrites
an existing LLM narrative — it falls back to preserving it. Detection: `model:` in markdown
frontmatter, or >500 chars with `## ` structural headings.

## Architecture Note: The Sync Gap

`quality-gate-swift` writes telemetry to disk but does not commit. `org-judgement-system`
has `CorpusManager.sync()` but it's only called from `ijs-telemetry push`, not from
`ijs-telemetry refine`. The generate-pulse.sh script now bridges this gap by running
`git add -A / commit / push` directly on the corpus directory before and after pulse
generation.
