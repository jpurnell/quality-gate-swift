## What Was Done

### 1. Dashboard Corpus Sync — Rebase Conflict Recovery (fix)

**Symptom**: `[dashboard] Warning: corpus pull failed (exit 1) — using local data`

**Investigation**: The org-judgement-corpus repo was stuck in `interactive rebase in progress`
on `pulse/2026-W21/PULSE_2026-W21.json`. Both local telemetry auto-commit and the remote
weekly pulse generation modified the same file, creating a real merge conflict.

**Root cause**: The previous recovery (added in `d420d90`) tried `git rebase --continue` on a
conflicted worktree — which always fails without manual conflict resolution. After aborting,
the code fell through to `pull --rebase` again, hitting the same conflict in a loop. This is
the second time this recovery logic has needed fixing.

**Fix** (`Dashboard.swift`, commit `bfb955d`):
1. Stuck rebase detected → abort immediately (don't attempt `--continue`)
2. `pull --rebase` conflict → abort that rebase, retry with `pull --no-rebase --strategy-option theirs`
3. Telemetry/pulse data is auto-generated, so remote wins on conflict — no data loss

**Resolution**: Accepted remote version of W21 pulse, pushed clean state to corpus.

### 2. IndexStorePass — CI Path Canonicalization (fix)

**Symptom**: `CrossModuleTests/flagsDeadInternal` failed on CI. The cross-module unreachable
code pass returned zero diagnostics — no skip note, no errors, just empty results.

**Investigation**: The test passes locally (arm64e, Swift 6.3) but fails on CI (macos-latest,
latest-stable Xcode). The syntactic pass (`unused_private`) worked fine on CI, meaning source
files were found and parsed. The cross-module pass ran without throwing (no `.note` diagnostics
for skip/stale), but `IndexStorePass.run()` returned an empty array. Traced through the 5-pass
pipeline:

- Pass 1 collects definitions via `db.symbols(inFilePath: file)` — if the index store records
  paths differently from SourceWalker (symlinks, normalization), this query returns empty
- Pass 5 has `if rec.fact == nil { continue }` — if `def.location.path` doesn't match the
  liveness index key, the fact lookup returns nil and the symbol is silently skipped
- `LivenessIndex` already had path canonicalization (`resolvingSymlinksInPath`) but
  `IndexStorePass` did not — every comparison used raw string equality

**Root cause**: Path mismatch between SourceWalker paths and IndexStoreDB paths on CI runners.
The entire pass silently returned empty because `db.symbols(inFilePath:)` found no matches.

**Fix** (`IndexStorePass.swift`, commit `e7f9365`):
1. Added cached `canonicalize()` via `resolvingSymlinksInPath()` applied to all path comparisons
2. Dual-path query: tries canonical path first, falls back to raw SourceWalker path for
   `db.symbols(inFilePath:)` in both Pass 1 and Pass 3
3. Canonicalized `def.location.path` and `occ.location.path` at comparison sites
4. Added `no_symbols` diagnostic hint when the pass finds zero indexed definitions despite
   having source files — makes future path issues visible instead of silently empty

All 63 UnreachableCodeAuditorTests pass locally. CI confirmation pending.

### 3. Pulse Narrative — LLM Generation Restored (fix, cross-repo)

**Symptom**: Thursday pulse narrative was thin/template-only despite previous weeks having rich
Claude-generated analysis.

**Investigation**: Compared W21 narrative frontmatter across commits. The initial corpus commit
(`2a99577`, May 18) had `model: claude-sonnet-4-6` in all narrative files (W16–W21) — full
LLM-generated analysis with lead signals, cluster deep-dives, and forward guidance. The weekly
`github-actions[bot]` commit (`c1cc108`, May 28) overwrote W20 and W21 with template output
(no `model:` field).

Tracked the generation pipeline:
- Workflow: `org-judgement-system/.github/workflows/weekly-pulse.yml`
- CLI: `ijs-telemetry refine` with `--narrate` flag
- Narrator: `ClaudeNarrativeSynthesizer` → `TemplateNarrativeSynthesizer` fallback
- Writer: `NarrativeWriter` produces markdown with frontmatter

**Root cause**: `--narrate` defaulted to `false` in `workflow_dispatch` inputs. Scheduled
Thursday cron runs never passed the flag, so the refine command always wrote template-only
narratives and overwrote existing LLM narratives.

**Fixes** (org-judgement-system, commit `fa4c475`):
1. `weekly-pulse.yml`: Scheduled runs always pass `--narrate`; manual dispatch default changed
   to `true`; `ANTHROPIC_API_KEY` env var wired to the generate step
2. `Refine.swift`: When `--narrate` is not passed, checks for existing LLM narrative (detected
   by `model:` in frontmatter) and preserves it in the pulse JSON instead of overwriting

**Additional** (org-judgement-corpus, commit `0fcd4d9`):
- Restored original LLM narratives for W20 and W21 from initial corpus commit
- `ANTHROPIC_API_KEY` secret added to org-judgement-system repo
